# ✨项目介绍

本项目是一个整合了([osu! - osu.ppy.sh](osu.ppy.sh))：
- `账户绑定`
- `比赛信息发布`
- `比赛信息查询`
- `比赛报名`
- `比赛中进程管理`
- `比赛回放文件整合`
- `比赛后奖励发放`
- `比赛信息统计`
等功能的综合osu!赛事平台。

*上述提到的比赛相关内容均为本项目合作方提供的，不代表osu!官方或其他组织*

# ✨核心用户角色
## 系统角色：
- **1级权限-普通用户**：普通用户
- **7级权限-内容审核人员**：负责审核用户发布的比赛信息、报名信息、奖励发放情况等，确保平台内容的质量和合法性。
- **8级权限-客服人员**：负责处理用户咨询、问题反馈、技术支持等，确保用户获得及时、优质的服务。
- **9级权限-管理员**：平台管理员，负责管理用户账号、审核内容、管理平台设置、管理内容审核人员、客服人员等。
- **9级权限-系统运维**：负责监控/维护平台的服务器、数据库、网络等基础设施，确保平台的正常运行和数据安全。

## 比赛角色：
- **5级权限-比赛组织者/比赛主办**：负责组织比赛、发布比赛信息、管理报名用户、管理奖品发放、协调比赛进程等。
- **4级权限-比赛选手**：报名参加比赛的用户，需要完成比赛要求的任务、上传比赛记录（一般为.osr文件）等。

# ✨概念解释
## 图池/图包/图：
1. 图池表示一个比赛的所有图的汇总，图池与比赛是 一对多 的关系
2. 图包表示包含一组图配置的包，图池与图包是 一对多 的关系，图包与图是 一对多 的关系
3. 图表示具体的osu谱面
## 【中文说明】页面名称（页面路径）
## `/console/*`：需要内容审核人员以上权限
## `/manage/*`：需要STAFF以上权限

# 技术栈
- 前端：Vue 3 + Nuxt 3 + Pania
- 后端：SpringBoot4 + WebSocket
- 数据库：MySQL + Redis
- 认证：JWT
